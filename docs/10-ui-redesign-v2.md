# 10-ui-redesign-v2.md — UI 视觉重构方案（v2 版）

## 1. 重构目标

基于参考设计图（`ui.png`）对 APP 进行全面 UI 升级，解决当前 UI "太 low" 的问题。核心变化：

- **色彩体系**：从 MD3 灰白红风格 → 暖调米灰背景 + 蓝色主色调 + 红色前区球 + 蓝色后区球
- **顶部导航**：新增蓝色渐变 Header 栏
- **底部导航**：从 emoji+指示条 → 圆角蓝色背景选中态
- **统计页**：从纯号码球展示 → 彩色水平柱状图
- **卡片样式**：增加柔和阴影/描边效果

## 3. 全局布局结构

```

├──────────────────────────────┤
│  ← 大乐透助手                │  ← Blue Header (56dp)
├──────────────────────────────┤
│                              │
│      Screen Content          │  ← 页面内容区 (Flexed 1)
│      (Bg: #E5DED2)          │

│  [Tab1]  [█Tab2█]  [Tab3]   │  ← Bottom Nav (64dp)
```

### 3.1 Header 组件

- 高度：56dp
- 背景色：`Primary` (`#176EFB`) 纯色
- 布局：`layout.Flex` 水平排列
  - 左侧：返回箭头/空白（当前无路由栈，可为空）
  - 中间/左侧：标题文字 "大乐透助手"（白色，粗体，20sp）
- 注意：gioui 不支持线性渐变，使用纯色 `Primary` 配合顶部高光（半透明白色弧）模拟

### 3.2 底部导航

每个 Tab 使用 `clip.RRect` 绘制整块圆角背景：

| 状态 | 背景 | 图标/文字色 |
|------|------|------------|
| 选中 | `NavSelectedBg` (`#D8E5FD`) | `Primary` (`#176EFB`) |
| 未选中 | 透明 | `Disabled` (`#8C8C8C`) |

导航栏高度 64dp，顶部 1px `Outline` 色分割线。

Tab 配置：

| 索引 | 图标 | 标签 | ScreenID |
|------|------|------|----------|
| 0 | 📋 | 开奖查询 | ScreenHistory |
| 1 | 📊 | 冷热统计 | ScreenStats |
| 2 | 🎯 | 智能推荐 | ScreenRecommend |

## 4. 开奖查询屏（History）

### 4.1 页面结构

```
┌─ Header ──────────────────────────────┐
│  开奖查询                        [刷新] │
└───────────────────────────────────────┘
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 期号    日期        前区        后区 │  │  ← 表头行（灰色背景，粘性顶部）
│  ├──────────────────────────────────┤  │
│  │ 24180  2026-07-09  05 12 18 23 31  07 11│  ← 数据行（白色背景）
│  │ 24179  2026-07-06  03 08 15 22 29  04 09│  ← 数据行（浅蓝交替色）
│  │ 24178  2026-07-02  01 10 19 28 35  02 12│  ← 数据行（白色背景）
│  │ ...                                   │
│  └──────────────────────────────────┘  │
│                                        │
│    ◀ 上一页          第 1/10 页        下一页 ▶ │
└────────────────────────────────────────┘
```

### 4.2 表格样式

- **表头行**：浅灰背景 `#F5F5F5`，粗体字 `#8C8C8C`，字号 11sp
- **数据行**：白色 `Surface` 与浅蓝 `#F0F4FA` 交替
- **行分隔线**：底部 1px `#E8ECF0` 色细线
- **列比例**：期号 18% / 日期 26% / 前区 34% / 后区 22%
- **前区号码**：`FrontBall` 色 `#F46E6A`（红色）
- **后区号码**：`BackBall` 色 `#2665EF`（蓝色）
- **表格不换行**：每行固定高度，内容超出时截断
- **单元格**：左对齐，左右 `th.Spacing.Small` 内边距

### 4.3 数据行格式

| 列 | 数据字段 | 格式 | 字体 |
|----|---------|------|------|
| 期号 | `draw.Issue` | 原始字符串，如 "24180" | Medium 12sp |
| 日期 | `draw.DrawTime` | `"2006-01-02"` | Normal 12sp |
| 前区 | `draw.FrontNumbers` | `"05 12 18 23 31"`（空格分隔，2位补零） | Medium 12sp |
| 后区 | `draw.BackNumbers` | `"07 11"`（空格分隔，2位补零） | Medium 12sp |

### 4.4 状态处理

| 状态 | 展示 |
|------|------|
| 加载中 | `LoadingSkeleton` 居中 |
| 错误 | "数据加载失败" + 错误详情 |
| 空数据 | "暂无开奖数据" |
| 正常 | 可滚动的表格（表头 + 数据行），每页 30 条

### 5.2 水平柱状图组件（新增）

**文件**：`ui/widget/chart.go`

**组件**：`HorizontalBarChart`

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `numbers` | `[]model.NumberFrequency` | 按频次降序排列的号码数据 |
| `barColor` | `color.NRGBA` | 柱状条颜色 |
| `maxFreq` | `int` | 本组最大频次（用于计算条宽比例） |

**渲染方式**：
- 每行包含：号码标签（左对齐）+ 彩色水平条 + 频次数值（右对齐）
- 条宽 = `(freq / maxFreq) * 可用宽度`
- 条高度 20dp，圆角 4dp
- 条颜色按热度/温度分档映射

**热度 → 颜色映射**：

| 分类 | 颜色 |
|------|------|
| 热号 (Hot) | `#FD9E37` (橙) |
| 温号 (Warm) | `#37C75F` (绿) |
| 冷号 (Cold) | `#2E70F9` (蓝) |
| 遗漏号 (Miss) | `#843CF8` (紫) |

### 5.3 滚动结构

使用 `layout.List` 垂直滚动，内容项：

| 索引 | 内容 |
|------|------|
| 0 | 前区号码热度卡片（含全部前区号码的水平柱状图） |
| 1 | 后区号码热度卡片（含全部后区号码的水平柱状图） |
| 2 | 底部免责声明 |

### 5.4 数据逻辑

维持现有 `service.Statistics.CalculateStats()` 不变，仅 UI 展示层从号码球切换为柱状图。
